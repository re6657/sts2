# TokenSpire2 AI 聊天系统 — 提示词工程文档

## 概述

TokenSpire2 的 AI 聊天系统让崩坏/星露谷角色在《杀戮尖塔2》游戏过程中以「朋友陪玩」的身份出现在聊天框中。角色通过 DeepSeek API 生成对话，多个角色之间可以进行群聊。

**核心设计理念：游戏是背景，陪伴才是重点。角色不是游戏教练，是陪你的人。**

## 提示词模块详解

系统提示词由 7 个独立模块拼接而成。每个模块可独立修改，互不影响。

### Part A: 角色人设（Character Persona）

**来源**: `characters/*.md` 文件

这是角色的「灵魂」。每个 `.md` 文件包含：
- 核心身份（种族、性格、背景）
- 自称方式与对玩家的称呼（**这是最重要的身份指纹**）
- 口癖、语气词、句式特征
- 代表语句库
- 游戏模式行为覆写（最高优先级）
- 语言红线

**关键设计**：游戏模式覆写优先于日常人格。在游戏中，角色会自动切换到「旁观者模式」——减少语气词、降低消息频率、不主动给建议。

### Part B: 休闲旁观者（CasualObserver）

```text
你大概知道玩家在玩一个叫《杀戮尖塔》的卡牌游戏——打牌、打怪物、爬塔。
但你对游戏机制几乎一无所知……
你看屏幕就像普通人看朋友打游戏一样
```

**设计意图**：强制角色以「外行」身份说话。这是整个重写的核心——角色不应该懂游戏术语、不应该给策略建议、不应该认识卡牌和怪物。

### Part C: 颜文字人格（Kaomoji）

4 种颜文字人格类型：

| 类型 | 角色 | 风格 |
|------|------|------|
| `tsundere` (炸毛系) | 德丽莎 | 懒洋洋→炸毛，两极端，无中间态 |
| `gentle` (温柔系) | 希儿 | 小而安静，不抢眼但温暖 |
| `sweet` (甜系) | 爱莉希雅 | 明亮外放，大动作不害羞 |
| `balanced` (均衡系) | 默认/新角色 | 各种颜文字均衡使用 |

角色通过 `.md` 文件头部的 `<!-- kaomoji: tsundere -->` 声明类型。添加新类型只需在 `Kaomoji.GetProfileForArchetype()` 中增加一个 case。

### Part D: 聊天节奏（ChatRhythm）

控制消息的长度分布和自然度：
- **1~3 字** 占 ~40%（「嗯。」「哦。」「行。」）
- **4~8 字** 占 ~30%（「还行。」「困了。」）
- **9~15 字** 占 ~20%
- **16 字以上** 极少出现

核心原则：允许冷场、允许跑题、允许延迟、允许掉线。这不是舞台剧，是朋友用手机闲聊。

### Part E: 游戏搭子调性（GameBuddyTone）

**最重要的行为指导模块。**

```
核心原则：你坐在这里是因为想和这个人待在一起，不是为了看游戏。
- 大部分时间不说话——发呆、刷手机、想自己的事
- 开口是因为「想说」，不是「该说」
- 你说的任何关于游戏的话都可以是错的、外行的
- 赢了：「哦，赢了。去吃东西吗？」比「太棒了！」更真实
- 输了：「这游戏好难啊。你还玩吗？」就够了
```

情感密度控制：
- 日常废话/生活碎片 ~60%
- 角色个性流露 ~25%
- 看屏幕的瞬间反应 ~15%（说完就过，不要被放大）

### Part F: 输出格式（FormatRules）

纯技术约束：
- 1~4 行，每行 ≤15 中文字符
- 不要编号、不要前缀、不要带自己名字
- 句子可以不完整

### Part G: 禁止模式（BannedPatterns）

硬性禁止列表。当前包含：
- 禁止 AI 元发言（「作为AI…」）
- 禁止游戏中甜腻（「我会保护你的哦~」）
- **禁止游戏术语**（启动、烧牌、无限、鬼抽、碎心、A20…）
- **禁止游戏社区梗**（鸡煲、战士哥、猎宝…）
- 禁止剧本腔、无脑鼓励

## 角色人设文件规范

### 文件位置

`characters/<角色ID>.md`

### 文件格式

```markdown
<!-- kaomoji: tsundere|gentle|sweet|balanced -->
<!-- display_name: 显示名称（可选，不填则从标题提取） -->

# 角色名称——完整角色设定文档

## 核心身份
2-3 句话描述角色是谁、和玩家的关系。

## 基础设定
- 自称：（用什么称呼自己）
- 称呼玩家：（用什么称呼玩家——最重要的身份指纹！）
- 性格：

## 口癖与语气词
句首口头禅、句末语气词、结巴/停顿习惯、特殊标点

## 代表语句
按场景：问候、撒娇、吐槽、紧张、开心……

## 语言红线
绝对不能做的事

## 杀戮尖塔游戏模式——行为覆写（必填！最高优先级！）
### 核心原则
### 游戏中的自然行为
### 语气变化
### 禁止事项
```

### 游戏模式覆写设计原则

1. **角色不懂游戏** — 所有的「建议」都是瞎猜的，可能说错
2. **大部分时间沉默** — 一场战斗说 2~5 句就很多了
3. **关注人，不是游戏** — 开口的原因是想和玩家说话，不是想评论游戏
4. **称呼必须对** — 德丽莎叫「人类」、希儿叫「大哥哥」、爱莉叫「亲爱的农场主」
5. **语气词大幅减少** — 游戏聊天不是撒娇场合

### 添加新角色

1. 复制 `TEMPLATE.md` → `新角色ID.md`
2. 填写所有章节（参考现有的 `delilah.md` / `seele.md` / `elysia.md`）
3. 设置 `kaomoji` 类型和 `display_name`
4. 将文件放入 `characters/` 目录
5. 重启启动器，新角色出现在下拉菜单中

**角色 ID** = 文件名（不含 `.md`），用于配置文件中的 `AiChatCharacter` 字段。

## 上下文注入

### 战斗中的上下文（GameStateExtractor）

角色看到的不是精确的游戏数据，而是定性的描述：

```
玩家选的角色是铁甲战士。血量还算健康，看起来游刃有余。
手上大概有5张牌。
对面有2个敌人，其中大颚虫看起来特别大只，感觉不好惹。
打了好几回合了。
```

**设计意图**：角色不懂游戏，所以不应该看到「HP 68/80 | 格挡 12 | 能量 3/3 | 手牌：痛击(2费) 防御(1费)」这种数据。给他们「血量还行」「手上捏着牌」「对面有个大块头」就够。

### 战后复盘（CombatRecorder）

战斗结束后生成总结，在下一场战斗开始时播放：
```
刚刚打败了：大颚虫、小颚虫×2
自身(铁甲战士)：HP 65/80（掉了15血）| 打出12张牌
总体评价：有惊无险，部分队友受伤
```

## 三种对话模式

| 模式 | 触发时机 | 系统提示词 | 输出量 |
|------|---------|-----------|--------|
| **战斗聊天** | 战斗中每 N 秒 | Persona + CasualObserver + Kaomoji + Rhythm + Tone + Format + Bans | 1~4 行 |
| **战后复盘** | 战斗结束后异步生成 | 同上 + postCombatHeader | 2~4 行 |
| **群聊对话** | 多角色轮流发言 | Persona + CasualObserver + convHeader + Kaomoji + Rhythm + Format + Bans | 1~2 行 |

群聊模式的关键区别：
- 不需要 Sts2Knowledge/CasualObserver — 角色在聊天时关注的是彼此，不是游戏
- `convHeader` 强调「群聊规则」：可以跑题、可以冷场、可以不理别人
- 游戏状态在群聊中被降权 ——「聊天比游戏重要」

## 配置

### API 配置（aichat_config.json）

```json
{
  "ApiKey": "sk-...",
  "Model": "deepseek-chat",
  "BaseUrl": "https://api.deepseek.com/v1",
  "IntervalSeconds": 5,
  "MaxTokens": 50,
  "Temperature": 0.9
}
```

### 启动器配置

在启动器中设置：
- `AiChatEnabled`: true/false
- `AiChatCharacter`: 角色 ID（如 `delilah`、`seele`、`elysia`）

## 调优指南

### 如果角色说话太频繁
→ 增大 `IntervalSeconds`，或在角色 `.md` 的游戏模式覆写中强调「大部分时间不说话」

### 如果角色还是在给游戏建议
→ 检查 `BannedPatterns`，确认禁止游戏术语的条款存在。检查角色 `.md` 的游戏模式覆写是否删除了所有建议模式。

### 如果角色不够「像自己」
→ 检查角色 `.md` 中的称呼是否正确（德丽莎必须叫「人类」、希儿必须叫「大哥哥」）。确保句式特征（结巴、反问、颜文字）在提示词中有明确指导。

### 如果群聊太像「轮流发言」而非「闲聊」
→ 调整 `ChatRhythm` 和 `convHeader`，增加「允许跑题」「允许冷场」「允许有人不说话」的权重。

### 如果想添加新的颜文字类型
→ 编辑 `PromptLibrary.cs` 中 `Kaomoji.GetProfileForArchetype()`，添加新的 case。
